# Приёмка в формате Gherkin

Feature: NFR-01 — Контроль доступа
  Scenario: Пользователь не может получить чужой список
    Given существует пользователь A и список пользователя B
    When пользователь A запрашивает GET /wishes/{id_B}
    Then сервер возвращает статус 404 NotFound

Feature: NFR-02 — Безопасное хранение паролей
  Scenario: Проверка хэширования паролей
    Given пользователь регистрируется с паролем "MySecure123!"
    When пароль сохраняется в базе
    Then он хранится в виде bcrypt-хэша

Feature: NFR-03 — Ограничение попыток логина
  Scenario: Блокировка после нескольких неверных попыток
    Given пользователь вводит неправильный пароль 5 раз подряд
    When происходит шестая попытка входа
    Then сервер возвращает ошибку 429 Too Many Requests

Feature: NFR-04 — Валидация входных данных
  Scenario: Попытка отправить неверный тип данных
    Given поле "price" ожидает число
    When клиент отправляет строку "abc"
    Then сервер возвращает 422 Unprocessable Entity

Feature: NFR-06 — Логирование событий безопасности
  Scenario: Проверка логирования неудачного логина
    Given пользователь вводит неверный пароль
    When сервер возвращает 401 Unauthorized
    Then событие записывается в файл security.log
